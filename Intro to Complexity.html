What is the Frequency Count Method?▶

Goal: Determine T(n), the total number of primitive operations a program executes as a function of input size n.

Method: Count how many times each statement executes, then sum all counts.

Counting Rules:
Assignment (x = 0;) → executes 1 time
Loop test (i < n) → executes n+1 times (n passes + 1 fail)
Loop body (inside for/while) → executes n times
Return → executes 1 time
Nested loop body → multiply outer × inner iterations
How to Analyze Loops▶

Ask three questions about any loop:

1. How does the loop variable change each iteration?
2. When does the loop stop? (What's the termination condition?)
3. How many iterations does that give as a function of n?

Understanding Asymptotic Notation▶
Big-O (Upper Bound): f(n) will never grow faster than c·g(n). Like a speed limit — you can go slower, but never faster.
Big-Ω (Lower Bound): f(n) will always grow at least as fast as c·g(n). Like a minimum speed — you're guaranteed to go at least this fast.
Big-Θ (Tight Bound): f(n) grows at exactly the same rate as g(n). Sandwiched between c₁·g(n) and c₂·g(n).

Relationships Between Notations

If f(n) = Θ(g(n)), then f(n) = O(g(n)) AND f(n) = Ω(g(n))
Θ is the intersection of O and Ω — both upper and lower bounds hold.
O gives the worst case guarantee. Ω gives the best case guarantee. Θ gives the exact growth rate.
